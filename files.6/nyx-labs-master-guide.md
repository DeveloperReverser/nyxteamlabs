# Nyx Labs — мастер-гайд: от нуля до сайта в топе

Один документ, по порядку. Делай шаги сверху вниз — каждый следующий зависит от предыдущего.

---

## ЧАСТЬ 0. Что вообще нужно завести (аккаунты)

Заведи заранее, займёт 15-20 минут суммарно:

- [ ] **GitHub** (github.com) — бесплатно, для кода и хостинга сайта
- [ ] **Render** (render.com) — бесплатно, для backend (server.py)
- [ ] **Telegram-бот** через **@BotFather** — для уведомлений и чата
- [ ] **Google Search Console** (search.google.com/search-console) — бесплатно
- [ ] **Яндекс.Вебмастер** (webmaster.yandex.ru) — бесплатно

Домен и хостинг сервера **покупать не нужно** — всё на бесплатных тарифах (детали и ограничения бесплатных тарифов — в Части 5).

## ЧАСТЬ 1. Файлы, которые у тебя уже готовы

- `index.html` — сам сайт (фронтенд, статика)
- `server.py` — backend: пересылка заявок в Telegram + живой чат
- `requirements.txt` — зависимости для server.py
- `robots.txt`, `sitemap.xml` — для поисковиков

Если потерял файлы — попроси меня прислать актуальную версию заново.

## ЧАСТЬ 2. Telegram-бот (нужен до деплоя backend'а)

1. В Telegram открой **@BotFather** → `/newbot` → задай имя и юзернейм → получишь **BOT_TOKEN** (длинная строка вида `123456:ABC-DEF...`).
2. Напиши своему новому боту любое сообщение ("привет").
3. Открой в браузере:
   `https://api.telegram.org/bot<ТВОЙ_ТОКЕН>/getUpdates`
4. В ответе найди `"chat":{"id": ЦИФРЫ ...}` — это **CHAT_ID**.
5. Придумай свой пароль — это будет **ADMIN_KEY** (для доступа к переписке в чате).

Сохрани все три значения — понадобятся в Части 4.

---

## ЧАСТЬ 3. Залить код на GitHub

1. На github.com → **+** → **New repository** → назови, например, `nyx-labs` → Create.
2. На странице репозитория → **Add file → Upload files** → закинь все 5 файлов (`index.html`, `server.py`, `requirements.txt`, `robots.txt`, `sitemap.xml`) → Commit changes.

(Если привычнее через терминал — `git init`, `git add .`, `git commit -m "init"`, `git remote add origin ...`, `git push`.)

---

## ЧАСТЬ 4. Деплой backend на Render

1. render.com → Sign up (лучше через GitHub).
2. **New → Web Service** → выбери репозиторий `nyx-labs` → Connect.
3. Настройки:
   - **Name**: `nyx-labs-backend` (или любое)
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn server:app --bind 0.0.0.0:$PORT`
   - **Plan**: Free
4. **Advanced → Add Environment Variable** — добавь три:
   - `BOT_TOKEN` = значение из Части 2
   - `CHAT_ID` = значение из Части 2
   - `ADMIN_KEY` = твой пароль из Части 2
5. **Create Web Service** → жди 2-5 минут.
6. Получишь адрес вида `https://nyx-labs-backend.onrender.com`. Открой его — должно показать `{"status":"Nyx Labs backend работает"}`.

---

## ЧАСТЬ 5. Обновить index.html реальными адресами

Открой `index.html`, замени плейсхолдеры (используй адрес из Части 4):

```js
const LEAD_ENDPOINT = 'https://nyx-labs-backend.onrender.com/api/lead';
const CHAT_ENDPOINT = 'https://nyx-labs-backend.onrender.com/api/chat';
```

Также замени `https://nyxlabs.example/` на реальный адрес сайта (получишь в Части 6) во всех meta-тегах (`canonical`, `og:url`, `og:image`, JSON-LD `url`), а также в `robots.txt` и `sitemap.xml`.

Загрузи обновлённый `index.html` обратно на GitHub (Upload files → заменить).

**Важные напоминания по безопасности:**
- Никогда не вписывай BOT_TOKEN/CHAT_ID/ADMIN_KEY прямо в код — только через переменные окружения на Render (Часть 4), особенно если репозиторий публичный.
- На бесплатном тарифе Render сервис засыпает без запросов на 15 минут (первый запрос после — 30-60 сек на "пробуждение"), а файлы на диске (включая базу чата) стираются при каждом перезапуске. Для теста — нормально. Для боевого режима с реальными клиентами — платный тариф ($7/мес) с постоянным диском снимает оба ограничения.

## ЧАСТЬ 6. Деплой сайта на GitHub Pages

1. В репозитории → **Settings → Pages**.
2. **Source**: Deploy from a branch → ветка `main`, папка `/ (root)` → Save.
3. Через 1-2 минуты получишь адрес: `https://ТВОЙ_НИК.github.io/nyx-labs/`.

Это твой рабочий сайт.

## ЧАСТЬ 8. Видимость в поиске (то, что реально работает)

**Важно держать в голове:** топ-1 по словам "разработка"/"команда" недостижим для нового сайта в принципе — не из-за настроек, а из-за конкуренции с агентствами с историей в десятки лет. Реальная цель — рост по конкретным длинным фразам, начиная с недель, а не "никогда".

### Сразу после деплоя
- [ ] Google Search Console и Яндекс.Вебмастер → добавить сайт → отправить `sitemap.xml`
- [ ] Яндекс Бизнес и 2ГИС → бесплатная регистрация (первая внешняя ссылка + локальная видимость)
- [ ] Профиль на GitHub/VC.ru/Habr со ссылкой на сайт

### Месяцы 2-3 — контент под конкретные фразы
Целимся не в "разработка", а в длинные фразы с низкой конкуренцией:
- разработка телеграм-бота с GPT
- парсер цен под заказ
- сайт для малого бизнеса под ключ
- AI-ассистент для поддержки клиентов
- автоматизация заявок в Telegram

Каждая фраза — заголовок будущей статьи. Лучшие статьи стоит со временем превратить в отдельные индексируемые страницы (сейчас они в JS-модалке и не видны поисковику) — это самый сильный технический рычаг на будущее.

- [ ] Отвечать на вопросы по теме на Habr Q&A / VC.ru / тематических Telegram-чатах со ссылкой на статью — естественные ссылки.

### Месяцы 4-6
- [ ] В Search Console/Вебмастере смотреть, по каким фразам уже есть показы — усиливать контент вокруг них.
- [ ] Заменить плейсхолдеры в портфолио/кейсах на реальные проекты — повышает конверсию тех, кто уже дошёл до сайта.

### Чего не делать
- **Не покупать ссылки, не накручивать показы/трафик** — за это банят, сайт может выпасть из индекса целиком.
- **Не обещать "топ-1 за неделю"** клиентам или себе — ни один честный метод этого не даёт.

## ЧАСТЬ 10 (опционально, позже). Свой домен

1. Купить домен (рег.ру/beget, ~150-200₽ первый год, ~400-1400₽/год продление).
2. Settings → Pages → Custom domain → впиши домен.
3. У регистратора в DNS добавь: 4 записи A на `185.199.108.153`, `.109.153`, `.110.153`, `.111.153`, и CNAME `www` → `ТВОЙ_НИК.github.io`.
4. Подожди обновления DNS (от минут до суток) → включи Enforce HTTPS в Settings → Pages.
